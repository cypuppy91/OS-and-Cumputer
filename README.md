# OS-and-Cumputer

# 💻 컴퓨터 공학 기초 학습 가이드

## 1. 컴퓨터 구조를 알아야 하는 이유
* **문제 해결 능력 향상:** 코드가 동작하지 않을 때, 컴퓨터를 두려운 미지의 대상이 아닌 분석의 대상으로 바라보고 근본적인 문제를 해결할 수 있는 능력을 기를 수 있습니다 [1, 2].
* **성능, 용량, 비용을 고려한 개발:** 클라우드 환경이나 서버 컴퓨터를 고를 때, 개발한 코드에 어느 정도의 성능과 용량이 필요한지 스스로 판단하고 비용을 최적화할 수 있습니다 [2-4].

## 2. 컴퓨터가 이해하는 정보와 4가지 핵심 부품
컴퓨터가 이해하는 정보는 크게 숫자, 문자, 이미지 같은 **데이터**(정적인 정보)와 컴퓨터를 실질적으로 움직이는 **명령어**로 나뉩니다 [5, 6].

### 🖥️ 4가지 핵심 부품
1. **메모리 (RAM):** 현재 실행 중인 프로그램의 명령어와 데이터를 저장하는 부품으로, 전원이 꺼지면 내용이 날아가는 휘발성 장치입니다 [7-9]. 데이터의 위치를 찾기 위해 '주소'라는 개념을 사용합니다 [10].
2. **CPU:** 메모리에 저장된 명령어를 읽어 들이고, 해석하고, 실행하는 컴퓨터의 두뇌입니다 [8].
   * **ALU (산술논리연산장치):** 실질적인 계산기 역할을 합니다 [11].
   * **레지스터:** CPU 내부에 있는 작은 임시 저장 장치로, 메모리보다 접근 속도가 훨씬 빠릅니다 [11, 12].
   * **제어장치:** 명령어를 해석하고 컴퓨터 부품들을 관리/작동시키기 위한 제어 신호를 보냅니다 [11, 13].
3. **보조기억장치 (SSD, HDD 등):** 전원이 꺼져도 지워지지 않아야 할 프로그램과 데이터를 보관하는 비휘발성 저장 장치입니다 [14].
4. **입출력장치 (마우스, 키보드, 모니터 등):** 컴퓨터 외부와 내부의 정보를 교환하는 부품입니다 [14].

* **메인보드와 시스템 버스:** 위 4가지 핵심 부품은 메인보드에 부착되어 연결되며, 부품 간 정보는 '시스템 버스'(주소 버스, 데이터 버스, 제어 버스)라는 핵심 통로를 통해 이동합니다 [15-17].

## 3. 데이터 표현 방식 (숫자와 문자)

### 🔢 숫자 표현
* **단위:** 0과 1을 표현하는 최소 단위인 비트(Bit)가 모여 정보의 단위가 되며, 8비트가 모이면 1바이트(Byte)가 됩니다 [18, 19].
* **이진법과 음수 표현:** 컴퓨터는 0과 1로 된 이진법을 사용합니다. 음수를 표현할 때는 모든 0과 1을 뒤집고 1을 더하는 '2의 보수법'을 주로 사용합니다 [20-22].
* **16진법 사용 이유:** 이진수는 숫자가 길어 사람이 읽기 힘들기 때문에, 코드나 주소를 표현할 때는 이진수와 변환이 아주 쉬운 16진법을 자주 사용합니다 [23-25].

### 🔠 문자 표현 (인코딩)
문자를 0과 1로 변환하는 과정을 **인코딩(Encoding)**, 그 반대를 **디코딩(Decoding)**이라고 합니다 [26, 27].
* **아스키코드 (ASCII):** 7비트를 사용하여 영어 알파벳과 특수문자 등 128개를 표현하지만, 한글은 표현할 수 없습니다 [28-30].
* **EUC-KR:** 한글 단어 하나하나에 고유한 코드를 부여하는 완성형 인코딩 방식입니다. 1글자당 2바이트를 차지하며 약 2,350자의 한글을 표현할 수 있으나, 모든 한글 조합(예: '뿩')을 표현하지는 못하는 한계가 있습니다 [31-34].
* **유니코드와 UTF-8:** 전 세계의 모든 언어, 특수문자, 이모티콘까지 포괄하는 통일된 문자 집합이 **유니코드**입니다 [35]. 이 유니코드 문자를 1~4바이트의 가변 길이로 변환하는 가장 대중적인 인코딩 방식이 **UTF-8**입니다 [36, 37].

## 4. 소스코드와 명령어, 그리고 컴파일 과정
개발자가 작성한 **고급 언어**(C, Python 등)는 컴퓨터가 이해하는 **저급 언어**(기계어, 어셈블리어)로 변환되어야 합니다 [38-40].

* **컴파일 언어 (예: C, C++):** 컴파일러가 소스 코드 전체를 쭉 훑어보고 한 번에 컴퓨터가 이해하는 목적 코드(기계어)로 변환합니다 [41, 42].
* **인터프리트 언어 (예: Python):** 인터프리터가 소스 코드를 한 줄씩 번역하며 즉시 실행합니다 [42, 43].

### ⚙️ C언어의 세부 컴파일 과정
C언어의 소스코드는 다음 4단계를 거쳐 실행 파일(.exe)이 됩니다 [44].
1. **전처리 (Preprocessing):** 외부 라이브러리 코드를 포함시키거나 매크로를 변환하는 등 컴파일 전 준비를 합니다 [44, 45].
2. **컴파일 (Compilation):** 전처리된 코드를 어셈블리어로 변환합니다 [46].
3. **어셈블 (Assembly):** 어셈블리어를 0과 1로 이루어진 기계어(목적 코드)로 변환합니다 [46, 47].
4. **링킹 (Linking):** 완성된 여러 개의 목적 코드들을 서로 연결하여 최종적으로 하나의 실행 파일로 만듭니다 [47].

## 5. 명령어의 구조와 주소 지정 방식
명령어는 수행할 동작을 뜻하는 **연산 코드**와, 연산에 사용할 데이터 또는 데이터의 위치가 담긴 **오퍼랜드(주소 필드)**로 구성됩니다 [48, 49].

명령어가 사용할 실제 데이터의 위치(유효 주소)를 찾는 방법을 **명령어 주소 지정 방식**이라고 합니다 [50].
* **즉시 주소 지정:** 오퍼랜드 필드에 직접 숫자를 써넣습니다. 메모리를 뒤질 필요가 없어 가장 빠릅니다 [51].
* **직접 주소 지정:** 오퍼랜드에 메모리 주소를 직접 적어, 그 주소로 찾아가 데이터를 가져옵니다 [52].
* **간접 주소 지정:** 오퍼랜드에 메모리 주소의 주소를 명시합니다. 표현할 수 있는 주소 공간은 넓어지지만, 메모리 접근이 여러 번 일어나 속도가 느립니다 [52, 53].
* **레지스터 주소 지정:** 연산할 데이터가 있는 레지스터를 명시합니다. CPU 내부의 레지스터를 사용하므로 메모리에 접근하는 것보다 훨씬 빠릅니다 [12, 53].

* # 💻 컴퓨터 공학 기초 학습 가이드 (심화편)

## 1. 정보의 단위와 음수 표현
컴퓨터는 0과 1로만 정보를 처리하며, 이진법을 기반으로 다양한 크기의 데이터를 다룹니다 [1, 2].

* **정보의 단위:** 0과 1을 표현하는 최소 단위인 비트(Bit)가 모여 바이트(Byte)가 되며, 바이트를 1,000개 단위로 묶어 킬로바이트(KB), 메가바이트(MB), 기가바이트(GB), 테라바이트(TB)로 부릅니다 [3]. (참고: 1,024개씩 묶는 단위는 KiB, MiB 등으로 엄밀히 구분하여 표기하는 추세입니다 [4, 5]).
* **워드(Word):** CPU가 한 번에 처리할 수 있는 정보의 크기를 의미합니다 [6]. 
* **음수 표현과 플래그:** 컴퓨터는 음수를 표현할 때 모든 0과 1을 뒤집고 1을 더하는 **'2의 보수법'**을 사용합니다 [7]. CPU는 메모리에 저장된 0과 1의 묶음이 양수인지 음수인지 구분하기 위해 내부의 **플래그(Flag) 레지스터**에 표시된 값을 확인합니다 [8].

## 2. 문자 인코딩의 한계와 발전
문자를 0과 1로 변환하는 인코딩 방식은 시대의 요구에 따라 발전해 왔습니다 [9, 10].

* **아스키코드 (ASCII):** 7비트를 사용하여 128개의 문자를 표현합니다 (나머지 1비트는 오류 검출용) [11, 12]. 알파벳과 기호는 표현 가능하지만 한글은 표현할 수 없습니다 [10].
* **EUC-KR의 한계:** 한글 글자 하나당 2바이트(16비트)를 부여하는 완성형 인코딩입니다 [13]. 약 2,350자의 한글을 표현할 수 있으나, '뿩'과 같은 일부 글자는 표현할 수 없어 이름이나 단어를 입력하지 못하는 문제가 발생하기도 했습니다 [14, 15].
* **유니코드 (Unicode)와 UTF-8:** 다국어 지원을 위해 전 세계 모든 문자에 고유한 16진수 값(코드 포인트)을 부여한 통일된 문자 집합이 유니코드입니다 [16, 17]. 이 유니코드 문자를 인코딩하는 대표적인 방식이 **UTF-8**이며, 글자의 종류(코드 포인트 범위)에 따라 1바이트에서 4바이트까지 크기가 조절되는 **가변 길이 인코딩** 방식을 취합니다 [18, 19].

## 3. 핵심 자료구조: 스택(Stack)과 큐(Queue)
데이터를 메모리에 저장하고 빼내는 대표적인 두 가지 구조입니다.

* **스택 (Stack):** 한쪽 끝이 막힌 통과 같은 구조입니다 [20]. 나중에 들어간 데이터가 가장 먼저 나오는 **LIFO(Last In First Out)** 방식으로 동작합니다 [21]. 데이터를 밀어 넣는 것을 푸시(Push), 빼내는 것을 팝(Pop)이라고 합니다 [20].
* **큐 (Queue):** 양쪽 끝이 뚫려 있는 파이프 같은 구조입니다 [22]. 먼저 들어간 데이터가 먼저 나오는 **FIFO(First In First Out)** 방식으로 동작합니다 [22].

## 4. 명령어의 연산 코드 종류
명령어는 '무엇을 할지'를 나타내는 **연산 코드**와 '어떤 데이터를 대상으로 할지'를 나타내는 **오퍼랜드**로 구성됩니다 [23]. 대표적인 연산 코드의 유형은 다음과 같습니다.

1. **데이터 전송:** 데이터를 옮기거나 메모리에 저장(STORE)하고, CPU로 불러옵니다(LOAD/FETCH) [24].
2. **산술/논리 연산:** 사칙연산, AND/OR/NOT 논리 연산 및 값의 크기를 비교합니다 [22].
3. **제어 흐름 변경:** 특정 메모리 주소로 실행 순서를 건너뛰거나(JUMP), 함수를 호출하고(CALL) 원래 위치로 되돌아옵니다(RETURN) [25].
4. **입출력 제어:** 외부 장치를 읽거나(READ) 쓰고(WRITE), 기기의 상태를 확인합니다(TEST) [26].

## 5. 시스템 버스의 세부 통로
메인보드에 연결된 4가지 핵심 부품(CPU, 메모리 등)은 **시스템 버스**라는 중추적인 통로를 통해 정보를 교환합니다 [27, 28]. 시스템 버스는 세 가지로 나뉩니다.

* **주소 버스:** 데이터를 읽거나 쓸 메모리의 특정 위치(주소)를 전달합니다 [29].
* **데이터 버스:** 실제 명령어와 데이터를 주고받는 통로입니다 [29].
* **제어 버스:** CPU가 메모리 읽기/쓰기 등 부품을 제어하기 위한 신호를 전달합니다 [29, 30].

## 6. C언어의 컴파일 과정 (심층 분석)
개발자가 작성한 고급 언어(소스 코드)가 컴퓨터가 이해하는 기계어로 변환되는 4단계 과정입니다 [31].

1. **전처리 (Preprocessing):** 본격적인 컴파일 전 준비 단계입니다. `#include` 같은 명령어를 통해 외부 라이브러리 소스 코드를 내 코드에 포함시키거나 매크로를 변환합니다 [31, 32].
2. **컴파일 (Compilation):** 전처리가 완료된 소스 코드를 컴퓨터가 이해할 수 있는 저급 언어인 **어셈블리어**로 번역합니다 [33]. (이때 CPU와 컴파일러 종류에 따라 어셈블리어의 형태가 달라집니다 [33].)
3. **어셈블 (Assembly):** 어셈블리어를 0과 1로 이루어진 기계어로 변환하여 **목적 파일(Object File)**을 생성합니다 [34].
4. **링킹 (Linking):** 완성된 목적 파일만으로는 실행이 불가능할 수 있습니다. 메인 코드와 외부 기능(예: 출력 함수)이 담긴 목적 코드들을 하나로 묶어 연결해주는 과정을 거쳐 최종 **실행 파일**이 완성됩니다 [34, 35].
